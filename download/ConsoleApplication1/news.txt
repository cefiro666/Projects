						<h1 id="internetreadfile-function" sourcefile="wininet/nf-wininet-internetreadfile.md" sourcestartlinenumber="45">InternetReadFile function</h1>

						<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr">
							<li class="displayDate loading">
								<time role="presentation" datetime="2018-12-05T00:00:00.000Z" data-article-date-source="ms.date">12/5/2018</time>
							</li>
								<li class="readingTime">2 minutes to read</li>

						</ul>

						<nav id="center-doc-outline" class="doc-outline is-hidden-desktop" data-bi-name="intopic toc" role="navigation" aria-label="On page navigation">
							<h3>In this article</h3>
						</nav>


						<!-- <content> -->
							
<p>Reads data from a handle opened by the
<a href="https://msdn.microsoft.com/73f969c3-3fa7-43f5-88c5-ba78e59a8d1c" data-linktype="external">InternetOpenUrl</a>,
<a href="https://msdn.microsoft.com/fb44d7bd-7868-4c53-aa4b-608d79c5bc7c" data-linktype="external">FtpOpenFile</a>,
or
<a href="https://msdn.microsoft.com/caaff8e8-7db9-4d6d-8ba2-d8d19475173a" data-linktype="external">HttpOpenRequest</a> function.</p>
<h2 id="syntax">Syntax</h2>
<pre><code class="lang-cpp">BOOLAPI InternetReadFile(
  HINTERNET hFile,
  LPVOID    lpBuffer,
  DWORD     dwNumberOfBytesToRead,
  LPDWORD   lpdwNumberOfBytesRead
);
</code></pre>
<h2 id="parameters">Parameters</h2>
<p><code>hFile</code></p>
<p>Handle returned from a previous call to
<a href="https://msdn.microsoft.com/73f969c3-3fa7-43f5-88c5-ba78e59a8d1c" data-linktype="external">InternetOpenUrl</a>,
<a href="https://msdn.microsoft.com/fb44d7bd-7868-4c53-aa4b-608d79c5bc7c" data-linktype="external">FtpOpenFile</a>,
or
<a href="https://msdn.microsoft.com/caaff8e8-7db9-4d6d-8ba2-d8d19475173a" data-linktype="external">HttpOpenRequest</a>.</p>
<p><code>lpBuffer</code></p>
<p>Pointer to a buffer that receives the data.</p>
<p><code>dwNumberOfBytesToRead</code></p>
<p>Number of bytes to be read.</p>
<p><code>lpdwNumberOfBytesRead</code></p>
<p>Pointer to a variable that receives the number of bytes read.
<b>InternetReadFile</b> sets this value to zero before doing any work or error checking.</p>
<h2 id="return-value">Return Value</h2>
<p>Returns <b>TRUE</b> if successful, or <b>FALSE</b> otherwise. To get extended error information, call
<a href="https://msdn.microsoft.com/d852e148-985c-416f-a5a7-27b6914b45d4" data-linktype="external">GetLastError</a>. An application can also use
<a href="https://msdn.microsoft.com/0aa274c5-0aa0-4eb9-8aef-3128e735759d" data-linktype="external">InternetGetLastResponseInfo</a> when necessary.</p>
<h2 id="remarks">Remarks</h2>
<p><b>InternetReadFile</b> operates much like the base
<a href="https://msdn.microsoft.com/4ad4580d-c002-44a4-a5f6-757e83ed8732" data-linktype="external">ReadFile</a> function, with a few exceptions. Typically,
<b>InternetReadFile</b> retrieves data from an
<a href="https://msdn.microsoft.com/8a9788ed-eb25-42cb-b912-8dffa3df1850" data-linktype="external">HINTERNET</a> handle as a sequential stream of bytes. The amount of data to be read for each call to
<b>InternetReadFile</b> is specified by the
<i>dwNumberOfBytesToRead</i> parameter and the data is returned in the
<i>lpBuffer</i> parameter. A normal read retrieves the specified
<i>dwNumberOfBytesToRead</i> for each call to
<b>InternetReadFile</b> until the end of the file is reached. To ensure all data is retrieved, an application must continue to call the
<b>InternetReadFile</b> function until the function returns <b>TRUE</b> and the
<i>lpdwNumberOfBytesRead</i> parameter equals zero. This is especially important if the requested data is written to the cache, because otherwise the cache will not be properly updated and the file downloaded will not be committed to the cache. Note that caching happens automatically unless the original request to open the data stream set the <b>INTERNET_FLAG_NO_CACHE_WRITE</b> flag.</p>
<p>When an application retrieves a handle using
<a href="https://msdn.microsoft.com/73f969c3-3fa7-43f5-88c5-ba78e59a8d1c" data-linktype="external">InternetOpenUrl</a>, WinINet attempts to make all data look like a file download, in an effort to make reading from the Internet easier for the application. For some types of information, such as FTP file directory listings, it converts the data to be returned by<br>
<b>InternetReadFile</b> to an HTML stream. It does this on a line-by-line basis. For example, it can convert an FTP directory listing to a line of HTML and return this HTML to the application.</p>
<p>WinINet attempts to write the HTML to the
<i>lpBuffer</i> buffer a line at a time. If the application's buffer is too small to fit at least one line of generated HTML, the error code
<b>ERROR_INSUFFICIENT_BUFFER</b> is returned as an indication to the application that it needs a larger buffer. Also, converted lines might not completely fill the buffer, so
<b>InternetReadFile</b> can return with less data in
<i>lpBuffer</i> than requested. Subsequent reads will retrieve all the converted HTML. The application must again check that all data is retrieved as described previously.</p>
<p>Like all other aspects of the WinINet API, this function cannot be safely called from within DllMain or the constructors and destructors of global objects.</p>
<p>When running asynchronously, if a call to <b>InternetReadFile</b> does not result in a completed transaction, it will return <i>FALSE</i> and a subsequent call to <a href="https://msdn.microsoft.com/d852e148-985c-416f-a5a7-27b6914b45d4" data-linktype="external">GetLastError</a> will return <i>ERROR_IO_PENDING</i>. When the transaction is completed the <a href="https://msdn.microsoft.com/a054fb71-66ab-46fd-be19-2237f05662bc" data-linktype="external">InternetStatusCallback</a> specified in a previous call to   <a href="https://msdn.microsoft.com/fe15627b-c77b-45c0-8ff6-02faa8512b57" data-linktype="external">InternetSetStatusCallback</a> will be called with <i>INTERNET_STATUS_REQUEST_COMPLETE</i>.</p>
<div class="alert"><b>Note</b>  WinINet does not support server implementations. In addition, it should not be used from a service.  For server implementations or services use <a href="https://msdn.microsoft.com/354ab65d-5e46-451d-b36b-2f8166a1a048" data-linktype="external">Microsoft Windows HTTP Services (WinHTTP)</a>.</div>
